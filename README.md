# Manual Testing Repository

## Overview
This repository contains manual testing artifacts including test plans, test cases, and test summary reports. It is designed to demonstrate practical knowledge of Software Quality Assurance (SQA) processes and documentation standards.

## Contents
- Test Plans: Defined testing scope, strategy, and approach for projects
- Test Cases: Detailed test scenarios with steps, expected results, and actual results
- Test Summary Reports: Execution results, defect summary, and overall testing status

## Purpose
The purpose of this repository is to showcase hands-on experience in manual testing and to maintain structured, reusable QA documentation aligned with industry practices.

## Skills Demonstrated
- Test case design and execution
- Requirement analysis
- Defect identification and reporting
- Test documentation and reporting

## Tools & Techniques
- Manual Testing
- Test Case Design Techniques (Equivalence Partitioning, Boundary Value Analysis)
- Documentation using standard QA templates
